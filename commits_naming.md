# Лучшие практики для именования коммитов в Git:

<тип>(<область>): <описание>

feat:     Новая функциональность
fix:      Исправление ошибки
docs:     Изменения в документации
style:    Форматирование, отсутствующие точки с запятой (без изменения кода)
refactor: Рефакторинг кода
test:     Добавление или исправление тестов
chore:    Обновление сборки, задачи, зависимостей
perf:     Изменения, улучшающие производительность
ci:       Изменения в CI/CD конфигурации
build:    Сборка проекта или изменения внешних зависимостей
