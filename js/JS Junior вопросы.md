# JavaScript: Краткие ответы на ключевые вопросы

## 1. Какие типы данных присутствуют в JavaScript?
**Примитивные (7 типов):**
- `number` — числа (целые, дробные, `Infinity`, `NaN`)
- `string` — строки
- `boolean` — `true` / `false`
- `undefined` — значение не присвоено
- `null` — "ничего", пустое значение
- `symbol` — уникальный идентификатор (ES6)
- `bigint` — целые числа произвольной длины (ES2020)

**Сложные (объектные):**
- `object` — объекты, массивы, функции, даты и т.д.

---

## 2. Что такое поднятие (hoisting) в JavaScript?
Механизм, при котором объявления переменных и функций "поднимаются" в начало своей области видимости перед выполнением кода.

- `var` — поднимается, но инициализируется `undefined`
- `let` / `const` — поднимаются, но находятся в TDZ до инициализации
- `function declaration` — поднимается полностью (можно вызвать до объявления)

---

## 3. Что делает отладчик?
Инструмент для пошагового выполнения кода, установки точек остановки (breakpoints), просмотра значений переменных и анализа стека вызовов. Позволяет находить и исправлять ошибки.

---

## 4. В чём разница между "==" и "==="?
- `==` — **абстрактное равенство**: выполняет приведение типов перед сравнением
- `===` — **строгое равенство**: сравнивает значения без приведения типов

```javascript
5 == "5"   // true (приведение)
5 === "5"  // false (разные типы)
```

---

## 5. В чём разница между var, let и const?
| Характеристика | `var` | `let` / `const` |
|---------------|-------|-----------------|
| Область видимости | функциональная | блочная (`{}`) |
| Поднятие | поднимается (значение `undefined`) | поднимается (TDZ) |
| Переобъявление | можно | нельзя |
| Повторное присвоение | можно | `let` — можно, `const` — нельзя |

---

## 6. Что такое свойство NaN в JavaScript?
`NaN` (Not-a-Number) — глобальное свойство, означающее "не число". Возникает при некорректных математических операциях.

**Особенности:**
- `NaN !== NaN` (оно не равно самому себе)
- Проверка: `isNaN()` или `Number.isNaN()`

---

## 7. Что такое каррирование (currying) в JavaScript?
Преобразование функции с несколькими аргументами в последовательность вложенных функций, каждая из которых принимает один аргумент.

```javascript
// Обычная функция
function sum(a, b, c) { return a + b + c; }

// Каррированная версия
function curriedSum(a) {
  return function(b) {
    return function(c) {
      return a + b + c;
    };
  };
}
curriedSum(1)(2)(3); // 6
```

---

## 8. Что такое прототипы объектов?
Механизм наследования в JavaScript. Каждый объект имеет скрытую ссылку `[[Prototype]]` (доступна через `__proto__` или `Object.getPrototypeOf()`), которая указывает на объект-прототип. При обращении к свойству, которого нет в объекте, поиск идет по цепочке прототипов.

---

## 9. Какие типы ошибок бывают в JavaScript?
- `SyntaxError` — синтаксическая ошибка
- `ReferenceError` — обращение к необъявленной переменной
- `TypeError` — операция с неподходящим типом
- `RangeError` — значение вне допустимого диапазона
- `URIError` — ошибки в функциях работы с URI
- `EvalError` — ошибки в `eval()` (редко)
- Пользовательские ошибки — через `throw` и наследование от `Error`

---

## 10. Что такое рекурсия?
Приём, при котором функция вызывает саму себя. Обязательно должно быть условие выхода (базовый случай), чтобы избежать бесконечного вызова.

```javascript
function factorial(n) {
  if (n <= 1) return 1; // базовый случай
  return n * factorial(n - 1); // рекурсивный вызов
}
```

---

## 11. Что такое DOM?
**DOM (Document Object Model)** — программный интерфейс для HTML/XML документов. Представляет структуру документа в виде дерева узлов, позволяя JavaScript динамически изменять содержимое, структуру и стили страницы.

---

## 12. В чём разница между клиентским и серверным JavaScript?
| Клиентский JavaScript | Серверный JavaScript |
|----------------------|---------------------|
| Выполняется в браузере пользователя | Выполняется на сервере (Node.js, Deno и др.) |
| Взаимодействует с DOM, событиями, `window` | Работает с файловой системой, БД, сетью |
| Доступ к API браузера | Доступ к серверным ресурсам |
| Пример: валидация форм, анимации | Пример: API-эндпоинты, работа с БД |

---

## 13. Что такое классы в JavaScript?
Синтаксический сахар (ES6) над прототипным наследованием. Позволяют создавать объекты с помощью более привычного ООП-синтаксиса.

```javascript
class Animal {
  constructor(name) {
    this.name = name;
  }
  speak() {
    console.log(`${this.name} издает звук`);
  }
}
```

---

## 14. Что такое функция-генератор?
Функция, которая может приостанавливать выполнение и возобновлять его. Объявляется `function*`, использует `yield` для возврата значения. Возвращает объект-итератор.

```javascript
function* generator() {
  yield 1;
  yield 2;
  yield 3;
}
const gen = generator();
gen.next(); // { value: 1, done: false }
```

---

## 15. Что такое WeakSet в JavaScript, и как он работает?
Коллекция, хранящая **только объекты** (не примитивы). Ссылки на объекты являются "слабыми" — если на объект нет других ссылок, он удаляется из WeakSet и сборщиком мусора.

**Особенности:**
- Нельзя итерировать
- Нет метода `size`
- Нет `forEach()`
- Методы: `add()`, `delete()`, `has()`

---

## 16. Разница между прототипическим и классическим наследованием
| Прототипическое (JS) | Классическое (Java, C#) |
|---------------------|------------------------|
| Объекты наследуются от объектов | Классы наследуются от классов |
| Цепочка прототипов | Иерархия классов |
| Гибкость, динамическое изменение прототипа | Строгая структура, проверка на этапе компиляции |
| В JS классы — это синтаксический сахар над прототипами | Классы — основа языка |

## 17. Что такое Temporal Dead Zone (TDZ) в JavaScript?
Временная мертвая зона — период с момента входа в блок кода до момента инициализации переменной, объявленной через `let` или `const`. В TDZ переменная существует, но недоступна (доступ вызывает `ReferenceError`).

```javascript
console.log(x); // ReferenceError (TDZ)
let x = 5;
```

## 18. Какие типы данных являются примитивными а какие сложными?
**Примитивные** (хранятся по значению, неизменяемы):

**Сложные** (хранятся по ссылке):
`object` (включая массивы, функции, даты, регулярные выражения и т.д.)

## 20. Что означает ключевое слово «this» в JavaScript?
`this` ссылается на контекст выполнения функции. Значение определяется тем, **как** вызвана функция:

- Обычный вызов → `window` / `global` (strict mode: `undefined`)
- Метод объекта → сам объект
- Конструктор (`new`) → новый объект
- `call` / `apply` / `bind` → переданный объект
- Стрелочная функция → наследует `this` из внешней области

## 21. Как создать объект в JavaScript?
```javascript
// Литерал
const obj1 = { name: "John" };

// Конструктор Object
const obj2 = new Object();

// Функция-конструктор
function Person(name) { this.name = name; }
const obj3 = new Person("John");

// Object.create()
const obj4 = Object.create(proto);

// Класс (ES6)
class Person { constructor(name) { this.name = name; } }
const obj5 = new Person("John");
```

## 22. Как создать массив в JavaScript?
```javascript
// Литерал
const arr1 = [1, 2, 3];

// Конструктор Array
const arr2 = new Array(5); // массив длиной 5 (пустые элементы)
const arr3 = new Array(1, 2, 3);

// Array.of() (ES6) — создает массив из аргументов
const arr4 = Array.of(5); // [5]

// Array.from() (ES6) — из итерируемого объекта
const arr5 = Array.from("hello"); // ['h','e','l','l','o']
```

## 23. Что такое callback-функция в JavaScript?
Функция, передаваемая в другую функцию в качестве аргумента и вызываемая после завершения определенной операции.

```javascript
function process(callback) {
  // выполнить операцию
  callback();
}

// Использование
process(() => console.log("Callback выполнен"));
```

Callback может быть:
- **Синхронным** — вызывается немедленно (например, `forEach`)
- **Асинхронным** — вызывается после события (например, `setTimeout`)

## 24. В чём разница между let и var?
| Характеристика | `var` | `let` |
|---------------|-------|-------|
| Область видимости | функциональная | блочная |
| Поднятие | поднимается (инициализируется `undefined`) | поднимается (TDZ) |
| Переобъявление в одной области | можно | нельзя |
| Глобальное объявление | создает свойство `window` | не создает свойство `window` |

## 25. Что такое замыкание в JavaScript?
**Замыкание** — это комбинация функции и лексического окружения, в котором она была объявлена. Внутренняя функция сохраняет доступ к переменным внешней функции даже после того, как внешняя функция завершила выполнение.

```javascript
function outer() {
  let counter = 0;
  return function inner() {
    counter++;
    return counter;
  };
}
const increment = outer();
increment(); // 1
increment(); // 2
// inner "замыкает" переменную counter
```

**Применение**: инкапсуляция данных, фабрики функций, мемоизация.
