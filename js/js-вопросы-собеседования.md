# Вопросы для собеседования
## **Классификация типов данных в JavaScript**

### **1. Примитивные (Primitive) типы (7 видов)**
Иммутабельные, хранятся по значению:
- **`string`** — строки: `"hello"`, `'world'`, `` `template` ``
- **`number`** — числа (целые, дробные, `Infinity`, `NaN`): `42`, `3.14`
- **`boolean`** — логические значения: `true`, `false`
- **`undefined`** — значение неопределенности (переменная объявлена, но не инициализирована)
- **`null`** — преднамеренное отсутствие значения (но typeof null → 'object' — это известная ошибка в JS)
- **`symbol`** (ES6) — уникальные идентификаторы: `Symbol('id')`
- **`bigint`** (ES2020) — большие целые числа: `123n`

### **2. Сложный типы**
Хранятся по ссылке, мутабельны:
- **`Object`** — базовый тип: `{ key: 'value' }`



## **Map — преобразует каждый элемент**
```javascript
[1, 2, 3].map(x => x * 2) // [2, 4, 6]
```
→ Новый массив той же длины

## **Filter — оставляет подходящие элементы**
```javascript
[1, 2, 3, 4].filter(x => x > 2) // [3, 4]
```
→ Новый массив (меньше или равен исходному)

## **Reduce — сводит к одному значению**
```javascript
[1, 2, 3].reduce((sum, x) => sum + x, 0) // 6
```
→ Любое значение (число, объект, массив)

---

### **Ключевое:**
- **Не меняют исходный массив** (чистые функции)
- **Принимают callback**: `(элемент, индекс, массив) => ...`
- **Цепочки**: можно комбинировать

### **Простое сравнение:**
- Нужно изменить данные → **map**
- Нужно отфильтровать → **filter**  
- Нужно получить одно значение → **reduce**

### **Пример цепочки:**
```javascript
[1, 2, 3, 4]
  .filter(x => x % 2 === 0)    // [2, 4]
  .map(x => x * 10)            // [20, 40]
  .reduce((a, b) => a + b, 0)  // 60

## **var** — устаревшее
```javascript
var x = 1; // устарело
```
- **Видна везде** в функции (function scope)
- Можно **переобъявлять**: `var x = 2;`
- Поднимается (hoisting) со значением `undefined`
- Присваивается к `window` (в браузере)

## **let** — для изменяемых переменных
```javascript
let y = 1;
y = 2; // OK
```
- **Блочная видимость** `{}` (block scope)
- Нельзя переобъявлять в одном блоке
- Можно менять значение
- Поднимается, но недоступна до объявления (временная мертвая зона)

## **const** — для констант
```javascript
const z = 1;
z = 2; // ❌ Ошибка!
```
- **Блочная видимость**
- Нельзя переобъявлять
- **Нельзя менять значение** (но объект/массив менять можно!)
```javascript
const arr = [1, 2];
arr.push(3); // ✅ OK
arr = [4, 5]; // ❌ Ошибка!
```

---

### **Краткое правило:**
1. **Всегда const** по умолчанию
2. **let** если нужно менять значение
3. **var** не использовать (кроме legacy кода)

### **Пример:**
```javascript
for (var i = 0; i < 3; i++) {} // i доступна после цикла
for (let j = 0; j < 3; j++) {} // j НЕ доступна после цикла
```

**Итог:** `let/const` → современно, предсказуемо, `var` → legacy

# Как нельзя называть имена переменных
Имена переменных **нельзя**:

- Начинать с **цифры** (`1name`)
- Содержать **пробелы** (`my name`)
- Использовать **спецсимволы** кроме `_` и `$` (`my-name`)
- Использовать **зарезервированные слова** (`let`, `const`, `if` и т.д.)
