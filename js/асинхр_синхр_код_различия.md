Тут важное уточнение: **асинхронная функция** в JavaScript — это не та, которая «запустится только после выполнения предыдущей», а та, которая **не блокирует** основной поток выполнения.

Давай разберем разницу на простых примерах.

---

### 1. Обычные (синхронные) функции — идут строго сверху вниз

Когда код синхронный, каждая следующая строка ждет завершения предыдущей.

```javascript
function first() {
  console.log("1. Начало first");
  for (let i = 0; i < 1e9; i++) {} // долгая работа
  console.log("2. Конец first");
}

function second() {
  console.log("3. second");
}

first();
second();
```

Вывод:
```
1. Начало first
2. Конец first
3. second
```
Всё последовательно. Пока `first` не закончит, `second` не начнется. Это **блокирующее выполнение**.

---

### 2. Асинхронная функция (`async/await` или колбэки/промисы)

Асинхронная функция **не ждет** внутри себя выполнения долгой операции, если та не асинхронная.  
Она может **приостановиться** только на `await`, но при этом не блокирует остальной код снаружи.

```javascript
async function asyncFirst() {
  console.log("1. Начало asyncFirst");
  await new Promise(resolve => setTimeout(resolve, 1000)); // ждем 1 сек
  console.log("4. Конец asyncFirst (после await)");
}

function second() {
  console.log("3. second");
}

asyncFirst();
second();
```

Вывод:
```
1. Начало asyncFirst
3. second
4. Конец asyncFirst (после await)
```

Обрати внимание:  
- `asyncFirst` начала выполняться  
- встретила `await` — отдала управление  
- выполнилась `second`  
- только потом вернулась и закончила `asyncFirst`

---

### 3. Главное отличие

| Обычная функция | Асинхронная функция |
|----------------|---------------------|
| Выполняется целиком сразу | Может приостанавливаться на `await` |
| Блокирует поток до завершения | Не блокирует поток (в местах ожидания) |
| Порядок вызова = порядок выполнения | Порядок выполнения может быть «перемешан» |

---

### 4. Почему путают с «запустится после предыдущей»

Возможно, ты имел в виду ситуацию, когда используют `await` **внутри** асинхронной функции:

```javascript
await asyncFirst(); // следующий код запустится только после asyncFirst
someFunction();
```

Но это не свойство асинхронной функции, а **поведение оператора `await`**, который заставляет ждать внутри той же функции.

Если же просто вызвать асинхронную функцию без `await`, она запустится, но не будет ждать завершения.
