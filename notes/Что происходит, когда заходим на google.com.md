# Что происходит, когда заходим на google.com
При заходе на `google.com` происходит следующее:

1. **DNS-запрос** — браузер спрашивает у DNS-сервера, какой IP-адрес у `google.com`.
2. **Установка соединения** — браузер подключается к этому IP через протокол TCP (порт 443 для HTTPS).
3. **TLS-рукопожатие** — браузер и сервер договариваются о шифровании (если сайт использует HTTPS, что для Google стандарт).
4. **HTTP-запрос** — браузер отправляет зашифрованный запрос на получение главной страницы (метод `GET /`).
5. **Ответ сервера** — Google возвращает HTML, CSS, JavaScript (обычно минимальную строку поиска и кнопки).
6. **Отображение** — браузер рендерит страницу с поисковой строкой и, если нужно, загружает дополнительные ресурсы (картинки, шрифты).

В итоге вы видите главную страницу Google.

## Что такое TCP, TLS и балансировщик нагрузки?

**TCP (Transmission Control Protocol)** — протокол, который устанавливает надёжное соединение между клиентом и сервером. Перед передачей данных происходит «трёхэтапное рукопожатие» (SYN → SYN-ACK → ACK), после которого гарантируется, что все пакеты дойдут в правильном порядке без потерь. Это фундамент любого интернет-соединения.

**TLS (Transport Layer Security)** — протокол шифрования, который надстраивается поверх TCP. Он превращает HTTP в HTTPS. В ходе TLS-рукопожатия стороны обмениваются сертификатами, подтверждают подлинность сервера и договариваются о сеансовых ключах шифрования. После этого весь трафик между браузером и сервером становится защищённым от перехвата и подмены.
---

**Аналогия для запоминания:**
> **TCP** — это надёжный курьер, который гарантирует, что посылка дойдёт целиком и в правильном порядке. **TLS** — это сейф-пакет, который никто не может вскрыть по пути.
