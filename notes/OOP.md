Вот кратко про объектно-ориентированное программирование (ООП) в JavaScript:

**ООП (Объектно-Ориентированное Программирование)** — это методология программирования, которая представляет программу как совокупность взаимодействующих объектов, 
каждый из которых является экземпляром определенного класса.

**Основная суть:** JavaScript — это язык, который поддерживает несколько парадигм, включая ООП. Его объектная модель основана на **прототипах**, а не на классических классах (как в Java или C++), хотя с появлением ES6 (2015) синтаксис стал выглядеть более привычно.

### Важный момент: что такое класс на самом деле?
**Класс в JavaScript — это функция.** Если проверить тип класса через `typeof`, мы получим `"function"`. Классы — это просто "синтаксический сахар" над функциями-конструкторами и прототипами.

```javascript
class MyClass {}
console.log(typeof MyClass); // "function"
```

Основные отличия класса от обычной функции:
- Класс нельзя вызвать без оператора `new` (будет ошибка)
- Методы класса являются неперечислимыми
- Код внутри класса всегда выполняется в строгом режиме

### 1. Ключевые концепции

*   **Объект:** Базовая единица, хранящая состояние (свойства) и поведение (методы). Создается с помощью фигурных скобок `{}`, конструктора `Object` или через класс.
    ```javascript
    const user = {
      name: 'Alex', // свойство
      age: 25,      // свойство
      greet() {     // метод
        console.log(`Привет, я ${this.name}`);
      }
    };
    ```

*   **Классы (ES6):** Синтаксический сахар над прототипной моделью. Удобный способ создания "шаблонов" для объектов.
    ```javascript
    class Car {
      constructor(brand, year) {
        this.brand = brand; // Поля (свойства)
        this.year = year;
      }
      
      startEngine() { // Метод класса
        console.log(`${this.brand} завелась!`);
      }
    }

    const myCar = new Car('Tesla', 2022); // Создание экземпляра
    ```

*   **Наследование (extends):** Возможность создавать классы на основе других, переиспользуя их код.
    ```javascript
    class ElectricCar extends Car {
      constructor(brand, year, batteryLife) {
        super(brand, year); // Вызов конструктора родителя (Car)
        this.batteryLife = batteryLife;
      }
      
      charge() {
        console.log(`Заряжаем батарею (осталось ${this.batteryLife}%)`);
      }
    }
    ```

*   **Инкапсуляция:** Скрытие внутренней реализации. В JS раньше использовали замыкания, сейчас активно применяется синтаксис с решеткой `#` (приватные поля).
    ```javascript
    class BankAccount {
      #balance = 0; // Приватное поле (недоступно снаружи)

      deposit(sum) {
        this.#balance += sum;
      }
    }
    ```

*   **Полиморфизм:** Способность объектов с одинаковым интерфейсом вести себя по-разному. (Например, метод `drive()` у автомобиля и у велосипеда).
    
    **Пример полиморфизма:**
    ```javascript
    // Базовый класс
    class Vehicle {
      drive() {
        console.log('Транспортное средство движется');
      }
    }
    
    // Разные реализации метода drive()
    class Car extends Vehicle {
      drive() {
        console.log('🚗 Автомобиль едет по дороге: врум-врум!');
      }
    }
    
    class Bicycle extends Vehicle {
      drive() {
        console.log('🚲 Велосипед крутит педали: скрип-скрип!');
      }
    }
    
    // Функция, использующая полиморфизм
    function startJourney(vehicle) {
      vehicle.drive(); // Один интерфейс — разное поведение
    }
    
    startJourney(new Car());    // 🚗 Автомобиль едет по дороге: врум-врум!
    startJourney(new Bicycle()); // 🚲 Велосипед крутит педали: скрип-скрип!
    ```

### 2. Главное отличие (Прототипы)
Вместо классов JS использует **прототипы**. Когда вы обращаетесь к свойству объекта, движок ищет его в самом объекте, а если не находит — идет по цепочке в его прототип (`__proto__` или `prototype` конструктора). Классы — это просто более элегантная обертка для этой механики.

### Итог
Современный JS для ООП использует **классы** и **конструкторы**, что делает код чистым и понятным, но под капотом по-прежнему работает мощная **прототипная** система.
