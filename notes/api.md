## Что такое API?

**API (Application Programming Interface)** — это набор правил и инструментов, с помощью которых одна программа может взаимодействовать с другой. Простыми словами, это «посредник», который позволяет приложениям обмениваться данными и функциями.

Пример: Когда вы заказываете пиццу через приложение, API доставляет ваш заказ на кухню и возвращает статус готовности.

## Основные виды API

### По доступности:
1. **Публичные (Open API)** — открыты для всех разработчиков (например, API Google Maps)
2. **Частные (Private API)** — используются только внутри компании
3. **Партнерские** — доступны только определенным партнерам

### По протоколам:
1. **REST API** — самый популярный, использует HTTP-запросы
2. **SOAP API** — строгий протокол на основе XML
3. **GraphQL** — позволяет запрашивать только нужные данные
4. **WebSocket** — для двусторонней связи в реальном времени

## Подробнее о протоколах API

### REST API
**Representational State Transfer** — это архитектурный стиль, а не строгий протокол. REST предоставляет стандартизированный способ предоставления данных другим приложениям. Затем эти приложения могут использовать данные, как им заблагорассудится. Иногда API также предлагают другим приложениям возможность вносить изменения в данные.

**Особенности:**
- Использует стандартные HTTP-методы (GET, POST, PUT, DELETE)
- Работает с данными в формате JSON (реже XML)
- Каждый ресурс имеет свой URL (эндпоинт)
- Не хранит состояние (stateless)

Обычно API — это окно в базу данных. Бэкенд API обрабатывает запросы к базе данных и форматирует ответ. Вы получаете статический ответ любого запрошенного вами ресурса, обычно в формате JSON.

**Пример:**
```
GET https://api.site.com/users/123 — получить пользователя
POST https://api.site.com/users — создать нового
```

### SOAP API
**Simple Object Access Protocol** — строгий протокол обмена сообщениями.

**Особенности:**
- Только XML-формат
- Имеет строгую структуру (Envelope, Header, Body)
- Встроенная обработка ошибок
- Поддерживает сложные операции и транзакции

**Где используется:** Банки, платежные системы, крупные корпорации (где важна надежность).

### WebSocket
Протокол для постоянного двустороннего соединения.

**Особенности:**
- Непрерывное соединение (не как HTTP запрос-ответ)
- Сервер может сам инициировать отправку данных
- Низкая задержка (low latency)

**Где используется:** Чаты, онлайн-игры, биржевые котировки, уведомления в реальном времени.

## Методы API (HTTP-методы)

В REST API используются стандартные HTTP-методы. Процесс запроса и преобразования значений табличной базы данных в JSON или другой формат называется **сериализацией**. При создании API правильная сериализация данных является основной проблемой.

| Метод | Действие | Описание |
|-------|----------|----------|
| **GET** | Получить данные | Самый распространенный вариант. Возвращает некоторые данные из API в зависимости от конечной точки, которую вы посещаете, и любых предоставленных вами параметров. |
| **POST** | Создать ресурс | Создает новую запись, которая добавляется в базу данных. |
| **PUT** | Полностью обновить | Ищет запись по указанному вами URI. Если она существует, обновляет существующую запись. Если нет, создает новую запись. |
| **PATCH** | Частично обновить | Обновляет отдельные поля записи. |
| **DELETE** | Удалить | Удаляет запись по заданному URI. |

### Коды ответов (status codes):
- **2xx** — успех (200 OK, 201 Created)
- **4xx** — ошибка клиента (404 Not Found, 400 Bad Request)
- **5xx** — ошибка сервера (500 Internal Server Error)

## Зачем это нужно?

Прежде чем мы перейдем к коду, стоит подумать, зачем вам создавать API. Если бы кто-нибудь объяснил мне эти основные концепции до того, как я начал, мне было бы намного лучше.

REST API настолько распространены в разработке программного обеспечения, что разработчику важно знать, как они работают. API — это то, как приложения взаимодействуют друг с другом или даже внутри себя.

Например, в веб-разработке многие приложения полагаются на REST API, чтобы интерфейсная часть могла взаимодействовать с серверной частью. Если вы развертываете приложение React поверх Django, вам понадобится API, чтобы React мог получать информацию из базы данных.
