## API: Кратко и просто

**API (Application Programming Interface)** — это посредник между программами. Как официант в ресторане: вы (клиент) говорите официанту (API), что хотите, он передает на кухню (сервер) и приносит вам заказ (данные).

### Суть
Программа отправляет **запрос** на чужой сервер и получает **ответ** (обычно в формате JSON). Это позволяет не изобретать велосипед, а использовать готовые функции других сервисов.

---

### Методы HTTP (глаголы REST)

Метод показывает, *какое действие* нужно выполнить с данными.

| Метод | Что делает | Аналог в жизни | Статус |
|:---|:---|:---|:---|
| **GET** | Получить данные | Посмотреть меню | Безопасно, можно повторять |
| **POST** | Создать новое | Сделать заказ | Изменяет данные |
| **PUT** | Полностью заменить | Переписать заново | Изменяет |
| **PATCH** | Частично изменить | Добавить соль | Изменяет |
| **DELETE** | Удалить | Вычеркнуть из списка | Изменяет |

### Пример запроса (GET)
```http
GET /users/123 HTTP/1.1
Host: api.site.com
```

### Пример ответа (JSON)
```json
{
  "id": 123,
  "name": "Анна",
  "age": 25
}
```

**Коротко о кодах ответа**
*   **2xx** — успех (200 OK, 201 Created)
*   **3xx** — перенаправление (ресурс временно или навсегда перемещен)
*   **4xx** — ошибка клиента (404 Not Found, 401 Unauthorized)
*   **5xx** — ошибка сервера (500 Internal Server Error)
